# Task 005: TON payment confirmation poller + status board

## Description

Poll the TON blockchain for incoming transfers that match payment requests. Update PaymentRequest status in Supabase. Build the live status board UI that shows paid / pending / overdue per participant.

## Acceptance Criteria
- [ ] `lib/rails/ton/poller.ts` — polls TON API for transfers to organizer address matching split IDs
- [ ] `app/api/payments/confirm/route.ts` — POST to manually confirm or receive webhook
- [ ] `app/api/splits/[id]/status/route.ts` — GET returns full status per participant
- [ ] `components/split-board.tsx` — live status board (paid ✅ / pending ⏳ / overdue ❌)
- [ ] Bot sends Telegram message when status changes
- [ ] Status updates within 10 s of on-chain confirmation
- [ ] Rail badge shown: "Paid via TON" or "Settled via XRPL"

## Technical Details

### TON polling strategy
Use TON Center API v2: `GET /getTransactions?address=<address>&limit=20`
Match by `comment` field containing `SatSplit:<splitId>:<participantId>`.

### Status board component
```tsx
<SplitBoard splitId={id}>
  <ParticipantCard name="Alice" status="paid" rail="ton" amount={18.75} />
  <ParticipantCard name="Bob" status="pending" amount={18.75} />
</SplitBoard>
```

### Supabase real-time
Subscribe to `payment_receipts` table changes for the split ID.
`const channel = supabase.channel('split-' + id).on('postgres_changes', ...)`
