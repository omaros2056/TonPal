# Task 002: Telegram bot + Mini App shell + webhook

## Description

Set up the grammy Telegram bot with all command handlers and photo message handler. Set up the Telegram Mini App shell (the page that opens when users tap the Mini App button in the bot). Wire the webhook endpoint.

## Acceptance Criteria
- [ ] `agents/telegram-bot/index.ts` — grammy bot instance with handlers
- [ ] `/start` command — welcome message + Mini App button
- [ ] `/split` command — starts a new split session
- [ ] Photo message handler — passes image to AI parse route, replies with summary
- [ ] Text message handler — natural language input (e.g. "dinner was €85 split 4 ways")
- [ ] `app/api/telegram/webhook/route.ts` — receives Telegram updates, delegates to bot
- [ ] `app/miniapp/page.tsx` — Mini App shell (renders inside Telegram)
- [ ] `app/miniapp/layout.tsx` — Telegram WebApp SDK init
- [ ] Bot responds correctly to all commands in local test

## Technical Details

### grammy bot structure
```typescript
import { Bot } from "grammy"
const bot = new Bot(process.env.TELEGRAM_BOT_TOKEN!)
bot.command("start", handleStart)
bot.command("split", handleSplit)
bot.on("message:photo", handlePhoto)
bot.on("message:text", handleText)
```

### Mini App init
```html
<script src="https://telegram.org/js/telegram-web-app.js"></script>
```
Use `window.Telegram.WebApp.ready()` on mount.

### Webhook route
```typescript
// POST /api/telegram/webhook
const { handleUpdate } = webhookCallback(bot, "std/http")
```
