# Task 006: ENS resolution + NameStone subnames + text records

## Description

Full ENS identity layer. Resolve `.eth` names to addresses + avatars. Create `alice.satsplit.eth` subnames gaslessly via NameStone. Store TON address and Telegram handle in ENS text records. Bridge ENS identity to TON payments.

## Acceptance Criteria
- [ ] `lib/ens/resolve.ts` — resolve ENS name → EVM address, avatar URL, display name
- [ ] `lib/ens/subnames.ts` — NameStone REST wrapper: create / update / delete subnames

- [ ] `app/api/ens/resolve/[name]/route.ts` — GET resolves name
- [ ] `app/api/ens/subnames/create/route.ts` — POST creates subname

- [ ] Feature flag: `ENS_ENABLED=false` skips all ENS calls gracefully

## Technical Details

### ENS resolution (viem)
```typescript
import { createPublicClient, http } from "viem"
import { mainnet } from "viem/chains"
import { normalize } from "viem/ens"

const address = await client.getEnsAddress({ name: normalize("alice.eth") })
const avatar = await client.getEnsAvatar({ name: normalize("alice.eth") })
const tonAddr = await client.getEnsText({ name: normalize("alice.eth"), key: "app.satsplit.ton-address" })
```

### NameStone subname creation
```typescript
// POST https://namestone.xyz/api/public_v1/set-name
{
  domain: "satsplit.eth",
  name: "alice",            // → alice.satsplit.eth
  address: "0x...",
  text_records: {
    "app.satsplit.ton-address": "UQ...",
    "app.satsplit.telegram": "@alice_tg"
  }
}
```

### ENS→TON bridge
When creating a PaymentRequest for an ENS participant:
1. Check `app.satsplit.ton-address` text record
2. If found, use as TON destination
3. If not, prompt user to connect TON wallet
