# Task 007: XRPL Checks — CheckCreate / Cash / Cancel + Xumm + poller

## Description

Full XRPL Checks integration. Participant commits to their share by creating an on-chain check. Organizer cashes all checks. Dispute flow: CheckCancel → Telegram notification. Status board shows XRPL rail alongside TON.

## Acceptance Criteria
- [ ] `lib/rails/xrpl/checks.ts` — CheckCreate / CheckCash / CheckCancel transaction builders
- [ ] `lib/rails/xrpl/xumm.ts` — Xumm sign request payload builder (QR + deeplink)
- [ ] `lib/rails/xrpl/poller.ts` — polls `/account_objects` for check state changes
- [ ] `app/api/xrpl/checks/create/route.ts` — POST builds check + returns Xumm payload
- [ ] `app/api/xrpl/checks/cash/route.ts` — POST cashes all pending checks for a split
- [ ] `app/api/xrpl/checks/status/[id]/route.ts` — GET returns check state
- [ ] `components/xrpl-check-flow.tsx` — shows Xumm QR code + sign status tracker
- [ ] Feature flag: `XRPL_ENABLED=false` skips all XRPL calls gracefully
- [ ] XRPL Testnet used for dev; demo-ready

## Technical Details

### CheckCreate transaction
```typescript
import { Client, CheckCreate } from "xrpl"
const tx: CheckCreate = {
  TransactionType: "CheckCreate",
  Account: participantXrpAddress,
  Destination: organizerXrpAddress,
  SendMax: xrpToDrops(amount.toString()),
  Expiration: unixTimeToRippleTime(Date.now() / 1000 + 7 * 24 * 3600), // 7 days
  InvoiceID: splitId + participantId, // for matching
}
```

### Xumm sign request
```typescript
// POST https://xumm.app/api/v1/platform/payload
{
  txjson: checkCreateTx,
  options: { expire: 10080 } // 7 days in minutes
}
// Returns: uuid, qr_png, deeplink
```

### Poller
Connect to XRPL Testnet WebSocket. Watch `account_objects` for organizer address. Match checks by InvoiceID. On state change → update Supabase → trigger Telegram notification.

### XRPL networks
- Dev: `wss://s.altnet.rippletest.net:51233`
- Demo: `wss://xrplcluster.com`
