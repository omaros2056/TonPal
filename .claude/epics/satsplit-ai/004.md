# Task 004: Split engine + participant assignment + TON payment links

## Description

Core split logic: create a split session, assign participants, calculate per-person amounts, generate TON payment links. This is the heart of the product.

## Acceptance Criteria
- [ ] `lib/split/engine.ts` — equal / itemized / custom split calculation
- [ ] `app/api/splits/route.ts` — POST creates split session in Supabase
- [ ] `app/api/splits/[id]/participants/route.ts` — POST adds participant
- [ ] `app/api/payment-links/route.ts` — POST creates PaymentRequest + TON deeplink
- [ ] `lib/rails/ton/payment-link.ts` — TON deeplink builder
- [ ] `components/participant-assignment.tsx` — add participants by Telegram handle, ENS name, or plain name
- [ ] `components/split-config.tsx` — choose split mode, adjust amounts
- [ ] Amounts always sum to total (validation enforced)
- [ ] Each participant gets a unique TON payment deeplink

## Technical Details

### TON deeplink format
```
ton://transfer/<address>?amount=<nanotons>&text=SatSplit:<splitId>:<participantId>
```

### Split engine
```typescript
function equalSplit(total: number, count: number): number[]
function itemizedSplit(items: ReceiptItem[], assignments: Assignment[]): Record<string, number>
function customSplit(amounts: Record<string, number>, total: number): void // validates sum
```

### PaymentRequest status flow
`pending` → `confirmed` | `overdue`
